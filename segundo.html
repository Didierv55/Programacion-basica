HTML

  • Escribe información pero no programa
  • Programar es cuando un código responde a una acción del usuario
  • <> = toda etiqueta HTML va entre los símbolos menor qué y mayor qué
  • <> = Toda etiqueta html debe cerrarse con el símbolo /
  • < title > = Es de mucha relevancia para los buscadores ya que es lo primero que indexan para el posicionamiento de una página
  • < body > = Contenido visual, qué realmente va dentro del navegador
  • Buenas prácticas = Estándar de manejar códiga para que cualquier programador pueda entenderlo fácilmente

CSS (Cascade Style Sheets)

  • Normalmente se agregan en un archivo externo o en < head >< style >AQUÍ VA EL CSS< /style >< /head >
  • Primero se agrega el ID qué es donde se va a aplicar el diseño
  • los atributos se encierran en llaves {}
  • Los atributos son las características de diseño que queremos que tenga nuestro contenido. Ej. Color de fondo, color de fuentes, sombras, etc.
  • Hay errores imperdonables como la falta de ; o de # en un código de color hexadecimal

JavaScript

  • Normalmente se agregan en un archivo externo o antes de terminar la etiqueta < body >< script>alert ("Aquí va el código JS");< /script>< /body>
  • Las variables de JS no permiten espacios se usa _ en nombres largos
  • Los números no pueden ir al principio de una variable, siempre debe ser una letra
  • No se pueden usar signos raros y diferencia mayúsculas y mínúsculas
  • Se puede agregar $ o _ antes de una variable y sigue funcionando (reglas raras de JS ;/)
  • = Buena práctica, especialmente en HTML5
  • < styletype="text/css"> = Buena práctica de CSS
  • < scripttype="text/javascript"> = Buena práctica de JS
